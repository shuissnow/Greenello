# インフラ設計書

## 構成概要

Docker Compose でフロントエンド・バックエンド・DB・Nginx の4コンテナを管理する。  
ローカル開発と AWS デプロイで同一の Docker 構成を使用し、環境差異を環境変数で吸収する。

## Docker 構成

| コンテナ名 | イメージ | 役割 | 内部ポート | 外部公開 |
|----------|---------|------|----------|---------|
| nginx | nginx:alpine | リバースプロキシ・静的ファイル配信 | 80 | 80 / 443 |
| frontend | node:20-alpine (ビルド成果物) | React SPA 静的ファイル（Nginx から配信） | — | — |
| backend | mcr.microsoft.com/dotnet/aspnet:8.0 | C# .NET Web API | 8080 | nginx 経由 |
| db | postgres:16-alpine | PostgreSQL データベース（ローカル開発用） | 5432 | 非公開 |

### Docker Compose ネットワーク

```
[nginx] ─────── [frontend (静的ファイルボリューム)]
    │
    └── app-network ── [backend] ── [db]
```

- `nginx` と `backend` は `app-network` で通信
- `db` は外部に直接公開しない
- `backend` から `db` へは `db:5432` で接続

### 主要ボリューム

| ボリューム名 | 用途 |
|-----------|------|
| postgres-data | PostgreSQL データ永続化（named volume） |

## 環境変数による切り替え

`.env` ファイルで環境差異を管理する。本番環境の機密情報はリポジトリに含めない。

| 変数名 | 説明 |
|-------|------|
| `DATABASE_URL` | PostgreSQL 接続文字列（ホスティング方式確定後に詳細定義） |
| `SESSION_SECRET` | セッション署名用シークレット |
| `SENDGRID_API_KEY` | SendGrid API キー |
| `FRONTEND_URL` | フロントエンドのオリジン URL（CORS 設定用） |
| `ASPNETCORE_ENVIRONMENT` | `Development` / `Production` |
