# API設計書

## 概要

エンドポイント定義・リクエスト/レスポンス仕様・エラーコードは OpenAPI スペックを正とする。
本書はスペックに表現できない設計方針・背景を記録する。

- **OpenAPI スペックファイル:** `backend/openapi.yaml`
- **Scalar UI:** 開発時は `GET /scalar/v1` で参照可能

---

## 設計方針

### ベース URL・バージョニング

- ベース URL は `/api/v1` で固定
- 破壊的変更が生じる場合のみ `/api/v2` に移行する（現時点では未定）

### 認証方式

- サーバーサイドセッション（HttpOnly Cookie）を採用
- JWT を使わない理由: 学習スコープでセッション無効化（ログアウト・アカウントロック）を確実に制御するため

### リソース設計の原則

- ボードリソースへのアクセスはリクエストユーザーがそのボードのメンバーであることを必須とする
- オーナー限定操作（ボード削除）は Service 層でチェックし、OpenAPI スペック上は `403` レスポンスを明示する

### 日時フォーマット

- すべて ISO 8601 形式（UTC）で返却する
- フロントエンド側でローカルタイムへの変換を行う

### エラーレスポンス

- 全エラーは `{ "error": { "code": "...", "message": "..." } }` の統一形式
- `code` は機械的に判定可能な文字列定数（例: `ACCOUNT_LOCKED`）とし、フロントエンドの分岐に使用する
