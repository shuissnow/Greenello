# バッチ・非同期処理設計書

## 処理一覧

| 処理名 | 種別 | トリガー | 概要 |
|-------|------|---------|------|
| 期限切れタスク通知ジョブ | 定期バッチ | 毎日 08:00 JST | 期限を過ぎた未完了タスクをメールで通知する |
| CSV エクスポート | 同期処理 | ユーザー操作（ボタンクリック） | フィルター条件に合うタスクを CSV で出力する |

> CSV エクスポートはリクエスト/レスポンス内で完結する同期処理（非同期キューは使用しない）。  
> 要件上の上限（1,000件・5秒以内）のためキューは不要と判断。

## 各処理の詳細

### 期限切れタスク通知ジョブ

- **トリガー**: .NET Hosted Service による定期実行（Cron 相当: 毎日 08:00 JST）
- **実装方法**: `IHostedService` / `BackgroundService` を継承したクラスで実装。`Timer` により 1 日 1 回実行
- **入力**: なし（DB から条件クエリで取得）

**通知対象の抽出条件**:
```
tasks.status IN ('inbox', 'in_progress')
AND tasks.due_date < CURRENT_DATE (JST)
AND tasks.due_date >= CURRENT_DATE - 30 days
AND tasks.due_date IS NOT NULL
```

**処理内容**:
1. 上記条件で tasks テーブルを SELECT
2. 各タスクについて、通知対象ユーザー（作成者・担当者）を特定
   - 作成者と担当者が同一ユーザーの場合は1通のみ
3. notification_logs テーブルで当日送信済みか確認（`WHERE task_id = ? AND user_id = ? AND sent_at = TODAY`）
4. 未送信のみ SendGrid API でメール送信
5. 送信成功後、notification_logs に INSERT

**メール本文の内容**:
- 件名: `【Greenello】期限切れタスクのお知らせ`
- 本文: タスクタイトル・期限日・ボード名・タスク詳細 URL

**出力**: notification_logs への INSERT・エラーログ（失敗時）

**エラー時の挙動**:
- 送信失敗時は最大 3 回リトライ（初回失敗から 5 分後・30 分後・1 時間後）
- 3 回のリトライを経ても失敗した場合はエラーログに記録（ユーザーへの再送は対象外）
- 個別タスクの送信失敗が他タスクの処理に影響しないよう、タスク単位でエラーハンドリングする

---

### CSV エクスポート

- **トリガー**: ユーザーが「CSV 出力」ボタンを押す（`GET /api/v1/boards/:boardId/tasks/export`）
- **入力**: ボードID・フィルター条件（クエリパラメータ）
- **処理内容**:
  1. ユーザー認証・ボード参加確認
  2. フィルター条件を基に tasks テーブルを SELECT（サブタスク含む）
  3. 結果をメモリ上で CSV に変換
  4. `Content-Type: text/csv; charset=utf-8` でレスポンス返却

**CSV 出力仕様**:

| カラムヘッダー | フィールド名 | 備考 |
|-------------|-----------|------|
| タスクID | tasks.id | |
| タイトル | tasks.title | |
| ステータス | tasks.status | `Inbox` / `進行中` / `完了` で出力 |
| 優先度 | tasks.priority | `高` / `中` / `低` / 空白（未設定） |
| カテゴリ | tasks.category | |
| 担当者 | users.email | 担当者のメールアドレス |
| 期限 | tasks.due_date | YYYY-MM-DD 形式 |
| 親タスクID | tasks.parent_task_id | サブタスクの場合に親タスクID、親タスクの場合は空白 |
| 作成日時 | tasks.created_at | ISO 8601 (JST) |
| 更新日時 | tasks.updated_at | ISO 8601 (JST) |

- 文字コード: UTF-8（BOM 付き、Excel での文字化け対策）
- 対象件数が 0 件の場合は API レベルで 400 エラーを返し、フロントエンドで警告表示

**出力**: CSV ファイル（ブラウザダウンロード）

**エラー時の挙動**:
- DB エラー等の予期せぬ失敗は 500 を返し、フロントエンドでエラートーストを表示
