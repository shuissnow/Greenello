# データフロー設計書

## データフロー概要

ユーザー操作はブラウザ上の React SPA が受け付け、REST API 経由でバックエンドに送信される。  
バックエンドはリクエストを検証し、Entity Framework Core を通じて PostgreSQL にデータを永続化し、レスポンスを返す。  
バックグラウンドジョブは非同期で DB を参照し、SendGrid 経由でメールを送信する。

## 主要フロー

### タスク作成

1. ユーザーが「タスク追加」ボタンを押し、タスク詳細モーダルを開く
2. フォームにタイトル・カテゴリ・期限・優先度・担当者を入力し「保存」
3. フロントエンドが `POST /api/v1/boards/:boardId/tasks` を送信
4. バックエンドがセッション認証・ボード参加確認・バリデーションを実施
5. tasks テーブルにレコードを INSERT（status = `inbox`）
6. 作成されたタスクオブジェクトを 201 で返却
7. フロントエンドが React Query のキャッシュを更新し、カンバン Inbox カラムに表示

### タスクステータス変更（Drag & Drop）

1. ユーザーがカンバン上のタスクカードをドラッグして別カラムにドロップ
2. フロントエンドがオプティミスティック更新（即時 UI 反映）
3. `PATCH /api/v1/tasks/:taskId` で `{ "status": "in_progress" }` を送信
4. バックエンドがセッション認証・ボード参加確認を実施
5. tasks テーブルの `status` を UPDATE
6. 更新されたタスクオブジェクトを 200 で返却
7. 失敗時はフロントエンドがオプティミスティック更新をロールバック

### CSV エクスポート

1. ユーザーがフィルター条件を指定し「CSV 出力」ボタンを押す
2. フロントエンドが `GET /api/v1/boards/:boardId/tasks/export?{filter params}` を送信
3. バックエンドがセッション認証・ボード参加確認を実施
4. フィルター条件を基に tasks テーブルを SELECT（サブタスク含む）
5. バックエンドがメモリ上で CSV を生成（UTF-8 BOM 付き）
6. `Content-Type: text/csv` でレスポンスを返却
7. ブラウザがファイルとしてダウンロード

### 期限切れタスク通知（バックグラウンドジョブ）

1. Hosted Service が毎日 08:00 JST に起動
2. tasks テーブルから条件に合うタスクを SELECT
   - status ≠ `done`
   - due_date < 当日
   - due_date ≥ 当日 - 30日
3. 通知対象ユーザー（作成者・担当者）を抽出
4. notification_logs テーブルで当日送信済みか確認（重複防止）
5. 未送信の場合、SendGrid API でメールを送信
6. 送信成功後、notification_logs にレコードを INSERT
7. 送信失敗時は最大3回リトライ（5分後・30分後・1時間後）。最終失敗はエラーログに記録

## 入出力定義

| 処理名 | 入力 | 出力 |
|-------|------|------|
| タスク作成 | タイトル（必須）・カテゴリ・期限・優先度・担当者ユーザーID | 作成されたタスクオブジェクト |
| タスク編集 | 更新フィールド（部分更新可） | 更新されたタスクオブジェクト |
| タスク削除 | タスクID | 204 No Content |
| ステータス変更 | タスクID・新ステータス | 更新されたタスクオブジェクト |
| タスク一覧取得 | ボードID・フィルター条件（任意） | タスク配列（サブタスク含む） |
| CSV エクスポート | ボードID・フィルター条件 | CSV ファイル（UTF-8 BOM）|
| ユーザー登録 | メールアドレス・パスワード | ユーザーオブジェクト（パスワードハッシュ除く） |
| ログイン | メールアドレス・パスワード | セッション Cookie |
| 期限切れ通知ジョブ | なし（定時実行） | 送信済み通知ログ・エラーログ |
