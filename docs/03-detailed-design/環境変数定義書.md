# 環境変数定義書

## 管理方針

- バックエンドの環境変数は `.env` ファイルで管理し、`docker-compose.yml` から読み込む
- `.env` ファイルはリポジトリに含めない（`.gitignore` に追加）
- リポジトリには `.env.example`（値なし）を含め、セットアップ手順に含める
- フロントエンドの環境変数は Vite の `VITE_` プレフィックス規則に従う

---

## バックエンド（ASP.NET Core）

| 変数名 | 型 | 必須 | 説明 | 例 | 環境 |
|--------|-----|------|------|-----|------|
| `ASPNETCORE_ENVIRONMENT` | string | Yes | 実行環境 | `Development` / `Production` | local / production |
| `ASPNETCORE_URLS` | string | Yes | バックエンドのリッスンURL | `http://+:8080` | local / production |
| `DATABASE_URL` | string | Yes | PostgreSQL 接続文字列 | `Host=db;Port=5432;Database=greenello;Username=postgres;Password=secret` | local / production |
| `SESSION_SECRET` | string | Yes | セッション署名用シークレットキー（32文字以上必須。起動時に32文字未満の場合は例外をスローする） | `your-random-secret-key-here` | local / production |
| `SESSION_TIMEOUT_MINUTES` | int | No | アイドルタイムアウト（分）。デフォルト: 30 | `30` | local / production |
| `SESSION_ABSOLUTE_TIMEOUT_DAYS` | int | No | 絶対タイムアウト（日）。デフォルト: 7 | `7` | local / production |
| `SENDGRID_API_KEY` | string | Yes | SendGrid API キー | `SG.xxxxxxxxxxxxxxxxxxxx` | local / production |
| `SENDGRID_FROM_EMAIL` | string | Yes | 送信元メールアドレス | `noreply@greenello.example.com` | local / production |
| `SENDGRID_FROM_NAME` | string | No | 送信元表示名。デフォルト: Greenello | `Greenello` | local / production |
| `FRONTEND_URL` | string | Yes | フロントエンドのオリジン URL（CORS 許可対象） | local: `http://localhost:5173` / prod: `https://greenello.example.com` | local / production |
| `NOTIFICATION_JOB_HOUR_JST` | int | No | 通知ジョブの実行時刻（JST 時間）。デフォルト: 8 | `8` | local / production |

---

## フロントエンド（Vite / React）

| 変数名 | 型 | 必須 | 説明 | 例 | 環境 |
|--------|-----|------|------|-----|------|
| `VITE_API_BASE_URL` | string | Yes | バックエンド API のベース URL | local: `http://localhost:8080/api/v1` / prod: `/api/v1` | local / production |

> 本番環境では Nginx がリバースプロキシを担うため、相対パス `/api/v1` を使用する。
>
> **注意:** Vite の `VITE_*` 変数はビルド時に静的に埋め込まれる。本番環境でリバースプロキシのパスを変更した場合は再ビルドが必要になる。本プロジェクトでは学習目的のためこの制約を許容し、ビルド時の静的埋め込み方式を採用する。

---

## Docker Compose 環境変数ファイル

ローカル開発用の `.env.example`:

```env
# Backend
ASPNETCORE_ENVIRONMENT=Development
ASPNETCORE_URLS=http://+:8080
DATABASE_URL=Host=db;Port=5432;Database=greenello;Username=postgres;Password=localpassword
SESSION_SECRET=change-this-to-a-random-32-char-string
SENDGRID_API_KEY=SG.your-api-key-here
SENDGRID_FROM_EMAIL=noreply@greenello.example.com
FRONTEND_URL=http://localhost:5173

# PostgreSQL (docker-compose db サービス用)
POSTGRES_DB=greenello
POSTGRES_USER=postgres
POSTGRES_PASSWORD=localpassword

# Frontend
VITE_API_BASE_URL=http://localhost:8080/api/v1
```

---

## セキュリティ上の注意

- `SESSION_SECRET` は本番環境では十分にランダムな値を生成する（例: `openssl rand -base64 32`）
- `SENDGRID_API_KEY` は SendGrid のダッシュボードで API キーの権限を「Mail Send」のみに制限する
- `DATABASE_URL` のパスワードは本番では強力なパスワードを使用する
- 本番環境の `.env` は EC2 インスタンス上に直接配置し、リポジトリには含めない
