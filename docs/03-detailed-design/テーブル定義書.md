# テーブル定義書

## テーブル一覧

| テーブル名 | 概要 |
|-----------|------|
| users | 登録ユーザー情報 |
| boards | ボード情報 |
| tasks | タスク・サブタスク情報 |
| notification_logs | 期限切れタスク通知の送信ログ |

---

## テーブル名: `users`

- **概要:** 登録ユーザーのアカウント情報・ログイン失敗カウント・ロック状態を管理する

| カラム名 | 型 | NULL | デフォルト | 制約 | 説明 |
|---------|-----|------|-----------|------|------|
| id | UUID | NO | gen_random_uuid() | PK | ユーザーID |
| email | VARCHAR(255) | NO | — | UNIQUE | メールアドレス |
| password_hash | VARCHAR(255) | NO | — | — | bcrypt ハッシュ |
| failed_login_count | INT | NO | 0 | — | 連続ログイン失敗回数 |
| locked_until | TIMESTAMPTZ | YES | NULL | — | ロック解除日時（NULL=非ロック） |
| created_at | TIMESTAMPTZ | NO | NOW() | — | 作成日時 |
| updated_at | TIMESTAMPTZ | NO | NOW() | — | 更新日時 |

- **インデックス:**
  | インデックス名 | カラム | 種別 |
  |--------------|--------|------|
  | users_pkey | id | PRIMARY KEY |
  | users_email_key | email | UNIQUE |

- **外部キー:** なし

- **ユーザー削除ポリシー:**
  - `boards.owner_user_id` と `tasks.created_by_user_id` が `RESTRICT` のため、ボードまたはタスクを持つユーザーは直接削除できない
  - ユーザー削除の事前条件: 対象ユーザーが所有するボード・作成したタスクをすべて削除してからユーザーを削除する
  - アプリケーション層で上記の順序を強制し、DB の RESTRICT 制約による外部キーエラーを防ぐ

## テーブル名: `boards`

- **概要:** ボードの基本情報を管理する。メンバー情報は board_members で管理

| カラム名 | 型 | NULL | デフォルト | 制約 | 説明 |
|---------|-----|------|-----------|------|------|
| id | UUID | NO | gen_random_uuid() | PK | ボードID |
| name | VARCHAR(100) | NO | — | — | ボード名 |
| owner_user_id | UUID | NO | — | FK(users.id) | オーナーのユーザーID |
| created_at | TIMESTAMPTZ | NO | NOW() | — | 作成日時 |
| updated_at | TIMESTAMPTZ | NO | NOW() | — | 更新日時 |

- **インデックス:**
  | インデックス名 | カラム | 種別 |
  |--------------|--------|------|
  | boards_pkey | id | PRIMARY KEY |
  | idx_boards_owner_user_id | owner_user_id | INDEX |

- **外部キー:**
  | カラム | 参照テーブル | 参照カラム | ON DELETE |
  |--------|------------|-----------|----------|
  | owner_user_id | users | id | RESTRICT |

---

## テーブル名: `tasks`

- **概要:** タスク・サブタスク情報を管理する。`parent_task_id` が NULL の場合が親タスク

| カラム名 | 型 | NULL | デフォルト | 制約 | 説明 |
|---------|-----|------|-----------|------|------|
| id | UUID | NO | gen_random_uuid() | PK | タスクID |
| board_id | UUID | NO | — | FK(boards.id) | 所属ボードID |
| parent_task_id | UUID | YES | NULL | FK(tasks.id) | 親タスクID（NULL=親タスク） |
| title | VARCHAR(100) | NO | — | — | タスクタイトル |
| status | VARCHAR(20) | NO | 'inbox' | CHECK(inbox/in_progress/done) | ステータス |
| priority | VARCHAR(10) | YES | NULL | CHECK(high/medium/low/NULL) | 優先度 |
| category | VARCHAR(20) | YES | NULL | — | カテゴリ（自由入力） |
| due_date | DATE | YES | NULL | — | 期限日（JST 基準の日付） |
| assignee_user_id | UUID | YES | NULL | FK(users.id) | 担当者ユーザーID |
| created_by_user_id | UUID | NO | — | FK(users.id) | 作成者ユーザーID |
| created_at | TIMESTAMPTZ | NO | NOW() | — | 作成日時 |
| updated_at | TIMESTAMPTZ | NO | NOW() | — | 更新日時 |

- **インデックス:**
  | インデックス名 | カラム | 種別 |
  |--------------|--------|------|
  | tasks_pkey | id | PRIMARY KEY |
  | idx_tasks_board_id | board_id | INDEX |
  | idx_tasks_parent_task_id | parent_task_id | INDEX |
  | idx_tasks_assignee_user_id | assignee_user_id | INDEX |
  | idx_tasks_status_due_date | (status, due_date) | INDEX（通知ジョブの抽出クエリ用） |

- **外部キー:**
  | カラム | 参照テーブル | 参照カラム | ON DELETE |
  |--------|------------|-----------|----------|
  | board_id | boards | id | CASCADE |
  | parent_task_id | tasks | id | CASCADE |
  | assignee_user_id | users | id | SET NULL |
  | created_by_user_id | users | id | RESTRICT |

---

## テーブル名: `notification_logs`

- **概要:** 期限切れタスク通知メールの送信履歴を管理する。同一タスク・ユーザー・日付の重複送信防止に使用

| カラム名 | 型 | NULL | デフォルト | 制約 | 説明 |
|---------|-----|------|-----------|------|------|
| id | UUID | NO | gen_random_uuid() | PK | ログID |
| task_id | UUID | NO | — | FK(tasks.id) | 通知対象タスクID |
| user_id | UUID | NO | — | FK(users.id) | 通知先ユーザーID |
| sent_at | DATE | NO | — | — | 送信日（JST 基準の日付） |
| created_at | TIMESTAMPTZ | NO | NOW() | — | 作成日時 |

- **インデックス:**
  | インデックス名 | カラム | 種別 |
  |--------------|--------|------|
  | notification_logs_pkey | id | PRIMARY KEY |
  | idx_notification_logs_task_user_date | (task_id, user_id, sent_at) | UNIQUE INDEX（重複送信防止） |

- **外部キー:**
  | カラム | 参照テーブル | 参照カラム | ON DELETE |
  |--------|------------|-----------|----------|
  | task_id | tasks | id | CASCADE |
  | user_id | users | id | RESTRICT |
