## Pages（レイヤー: ルート対応・レイアウト組み立て）

### RegisterPage

- **概要:** 新規登録画面のレイアウトコンポーネント
- **パス:** `/register`
- **責務:** RegisterForm のレンダリング、未認証ユーザー専用（ログイン済みはホームへリダイレクト）
- **依存コンポーネント/フック:** RegisterForm, useCurrentUser

### LoginPage

- **概要:** ログイン画面のレイアウトコンポーネント
- **パス:** `/login`
- **責務:** LoginForm のレンダリング、未認証ユーザー専用
- **依存コンポーネント/フック:** LoginForm, useCurrentUser

### HomePage

- **概要:** ボード一覧のホーム画面
- **パス:** `/`（認証必須）
- **責務:** ボード一覧の取得・表示、ボード作成モーダルの開閉制御
- **内部状態:**
  | 状態名 | 型 | 説明 |
  |--------|-----|------|
  | isNewBoardModalOpen | boolean | 新規ボード作成モーダルの開閉 |
- **依存コンポーネント/フック:** GlobalNav, BoardList, NewBoardModal, useBoards

### BoardPage

- **概要:** ボード画面のレイアウト。URL の `view` パラメータによりビューを切り替える
- **パス:** `/boards/:boardId`（認証必須）
- **責務:** フィルター状態（URL クエリパラメータ）管理、ビュー切替、タスク詳細モーダル開閉制御
- **内部状態:**
  | 状態名 | 型 | 説明 |
  |--------|-----|------|
  | selectedTaskId | string \| null | タスク詳細モーダルで表示中のタスクID |
  | isNewTaskModalOpen | boolean | 新規タスク作成モーダルの開閉 |
- **依存コンポーネント/フック:** GlobalNav, ViewTabs, FilterPanel, KanbanBoard, TaskListView, CalendarView, SummaryView, TaskDetailModal, useBoard, useTasks, useSearchParams

### BoardSettingsPage

- **概要:** ボード設定画面（ボード削除）
- **パス:** `/boards/:boardId/settings`（認証必須・オーナーのみ）
- **責務:** ボード削除の確認 UI
- **依存コンポーネント/フック:** GlobalNav, useBoard

### AccountPage

- **概要:** アカウント設定画面（パスワード変更）
- **パス:** `/account`（認証必須）
- **責務:** PasswordChangeForm のレンダリング
- **依存コンポーネント/フック:** GlobalNav, PasswordChangeForm

## Hooks（レイヤー: サーバー状態管理・API 呼び出し抽象化）

TanStack Query を使用してサーバー状態を管理する。各フックのキャッシュキーは下記に定義する。

### useCurrentUser

- **概要:** ログイン中ユーザー情報の取得・ログイン・ログアウト
- **依存モジュール:** api/auth.ts, QueryClient (TanStack Query)
- **クエリキー:** `['currentUser']`
- **キャッシュ戦略:** staleTime: Infinity（明示的な invalidate まで再フェッチしない）
- **提供する操作:**
  | 操作 | 種別 | 処理概要 |
  |------|------|---------|
  | `currentUser` | Query | GET /auth/me でログイン中ユーザーを取得（401 の場合は null） |
  | `login(email, password)` | Mutation | POST /auth/login → 成功時に `['currentUser']` を invalidate |
  | `logout()` | Mutation | POST /auth/logout → 成功時に全クエリをリセット |

---

### useBoards

- **概要:** ボード一覧の取得・作成
- **依存モジュール:** api/boards.ts
- **クエリキー:** `['boards']`
- **提供する操作:**
  | 操作 | 種別 | 処理概要 |
  |------|------|---------|
  | `boards` | Query | GET /boards でボード一覧を取得 |
  | `createBoard(name)` | Mutation | POST /boards → 成功時に `['boards']` を invalidate |

---

### useBoard(boardId)

- **概要:** 特定ボードの詳細取得・ボード削除
- **依存モジュール:** api/boards.ts
- **クエリキー:** `['boards', boardId]`
- **提供する操作:**
  | 操作 | 種別 | 処理概要 |
  |------|------|---------|
  | `board` | Query | GET /boards/:boardId でボード詳細を取得 |
  | `deleteBoard()` | Mutation | DELETE /boards/:boardId → 成功時に `['boards']` を invalidate しホームへリダイレクト |

---

### useTasks(boardId, filters)

- **概要:** タスク一覧の取得・作成・更新（オプティミスティック）・削除・サブタスク作成
- **依存モジュール:** api/tasks.ts
- **クエリキー:** `['tasks', boardId, filters]`
- **提供する操作:**
  | 操作 | 種別 | 処理概要 |
  |------|------|---------|
  | `tasks` | Query | GET /boards/:boardId/tasks でタスク一覧を取得 |
  | `createTask(input)` | Mutation | POST /boards/:boardId/tasks → 成功時に `['tasks', boardId]` を invalidate |
  | `updateTask(taskId, input)` | Mutation | PATCH /tasks/:taskId → 成功時に `['tasks', boardId]` と `['task', taskId]` を invalidate |
  | `updateTaskStatusOptimistic(taskId, newStatus)` | Mutation | オプティミスティック更新（下記参照） |
  | `deleteTask(taskId)` | Mutation | DELETE /tasks/:taskId → 成功時に `['tasks', boardId]` を invalidate |
  | `createSubtask(parentTaskId, title)` | Mutation | POST /tasks/:taskId/subtasks → 成功時に `['task', parentTaskId]` を invalidate |

- **Optimistic Update パターン（updateTaskStatusOptimistic）:**
  ```
  onMutate: (taskId, newStatus):
    // 1. 進行中のリフェッチをキャンセル（競合防止）
    queryClient.cancelQueries(['tasks', boardId])
    // 2. 現在のキャッシュを保存（ロールバック用スナップショット）
    snapshot = queryClient.getQueryData(['tasks', boardId])
    // 3. キャッシュを即時書き換え（UI に即反映）
    queryClient.setQueryData(['tasks', boardId], tasks =>
      tasks.map(t => t.id === taskId ? { ...t, status: newStatus } : t)
    )
    return { snapshot }

  onError: (error, variables, context):
    // 4. 失敗時: スナップショットに戻す
    queryClient.setQueryData(['tasks', boardId], context.snapshot)
    // 5. エラートーストを表示
    showToast('ステータスの更新に失敗しました', 'error')

  onSettled:
    // 6. 成否に関わらずサーバーデータで再同期
    queryClient.invalidateQueries(['tasks', boardId])
  ```
