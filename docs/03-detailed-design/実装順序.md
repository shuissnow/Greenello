# Greenello 実装順序

詳細設計書を基に、機能間の依存関係を整理した実装順序リスト。
`/plan-task` スキルで作業単位を選ぶ際の指針として使用する。

---

## 技術スタック

- **Backend:** C# .NET 8 Web API + EF Core + PostgreSQL 16
- **Frontend:** React 18 + Vite + TypeScript + TanStack Query + dnd-kit + Tailwind CSS
- **Email:** SendGrid
- **Infra:** Docker Compose + Nginx + AWS

---

## 実装順序リスト

依存関係に従い、上から順に実装する。

| # | 作業単位 | スコープ | 説明 |

## 依存関係

```
Phase 1（インフラ）
  └─ Phase 2-BE（認証バックエンド）
       └─ Phase 3-BE（ボードバックエンド）
            └─ Phase 7-BE（タスクバックエンド）
                 ├─ Phase 6（CSVエクスポート）
                 └─ Phase 8-BE（通知バッチ）

  └─ Phase 2-FE（認証フロントエンド）
       └─ Phase 3-FE（ホーム画面）
            └─ Phase 4-FE（カンバンビュー）
                 ├─ Phase 5-FE-9（リストビュー）
                 ├─ Phase 5-FE-10（カレンダービュー）
                 ├─ Phase 5-FE-11（サマリービュー）
                 └─ Phase 9（ボード設定画面）
```

## 使い方

1. このリストで実装する番号を選ぶ
2. `/plan-task` スキルを実行して作業単位ドキュメントを作成する
3. `/implement-code` スキルで実装を開始する
