# ドキュメント作業ルール

## 文体ルール

- 日本語のです/ます調で記述してください
- 技術用語（API・CRUD・SPA・JWT など）は英語表記で構いません
- 略語を初出で使う場合は日本語説明を併記してください

## ステータス凡例

ドキュメント内の各項目には以下のステータスタグを付与してください。

| タグ | 意味 | Claude の扱い |
|-----|------|--------------|
| `[確定]` | 設計が確定済みです | 変更する場合は必ずユーザーに確認してください |
| `[検討中]` | 未確定・議論中です | 実装・下流ドキュメントへの転記前に必ずユーザーに確認してください |
| `[TBD]` | 後続フェーズで決定予定です | 現時点ではスキップし、決定後に追記してください |

タグなしの記述は `[確定]` と同等に扱ってください。

## ドキュメント間の依存ルール

- 上流ドキュメントで `[検討中]` または `[TBD]` の項目は、下流ドキュメントに転記しないでください
- 転記が必要な場合は同じタグを付与し、確定前であることを明示してください

```
docs/01-requirements/  ←  上流（要件定義）
      ↓
docs/02-basic-design/  ←  中流（基本設計）
      ↓
docs/03-detailed-design/ ←  下流（詳細設計）
```

## 成果物一覧の更新ルール

- ドキュメントを追加・削除・リネームした場合は、同ディレクトリの `*成果物一覧.md` を必ず同時に更新してください

## 開発フロー

このファイルで定義するのはドキュメント作業フェーズのみです。実装フェーズ以降のルールは `src/CLAUDE.md` を参照してください。

各ステップで対応するスキルを呼び出してください。

```
要件定義     /design-requirements-doc
  ↓ レビュー  /review-docs
基本設計     /design-basic-doc
  ↓ レビュー  /review-docs
詳細設計     /design-detailed-doc
  ↓ レビュー  /review-docs
（実装フェーズ以降は src/CLAUDE.md を参照してください）
```

# ドキュメント構成

| ディレクトリ                | 用途                                             |
| --------------------------- | ------------------------------------------------ |
| `docs/00-ideas/`            | アイデアメモや初期構想を格納します               |
| `docs/01-requirements/`     | 要件定義書を格納します                           |
| `docs/02-basic-design/`     | 基本設計書を格納します                           |
| `docs/03-detailed-design/`  | 詳細設計書を格納します                           |
| `docs/04-implementation/`   | 機能単位の実装計画やタスクリストを格納します     |
| `docs/05-unit-test/`        | 単体テスト関連ドキュメントを格納します           |
| `docs/06-integration-test/` | 結合テスト関連ドキュメントを格納します           |
| `docs/07-system-test/`      | システムテスト関連ドキュメントを格納します       |
