# 業務フロー図

## As-Is（現状フロー）

ツールなし・スプレッドシート・付箋など個人ごとに異なる手段でタスクを管理している状態。

```mermaid
flowchart TD
    A[タスクが発生] --> B[スプレッドシートや付箋にメモ]
    B --> C{チームで共有する?}
    C -->|はい| D[メール・チャットで共有]
    C -->|いいえ| E[個人で管理・追跡]
    D --> F[各自がコピーして管理]
    F --> G[進捗を口頭・メールで確認]
    E --> G
    G --> H{完了?}
    H -->|はい| I[手動でステータス更新]
    H -->|いいえ| G
    I --> J[期限切れに気づかない場合あり]
```

## To-Be（あるべき姿）

Greenello を使い、チームが一つのボードでタスクを共同管理する状態。

```mermaid
flowchart TD
    A[タスクが発生] --> B[Greenello でタスクを作成]
    B --> C[カンバン / リスト / カレンダーで即時参照]
    C --> D{担当者を設定する?}
    D -->|はい| E[担当者をアサイン]
    D -->|いいえ| F[未アサインのまま管理]
    E --> G[カンバン / リスト / カレンダーで進捗を確認]
    F --> G
    G --> H[Drag & Drop でステータスを更新]
    H --> I{完了?}
    I -->|はい| J[完了ステータスへ移動]
    I -->|いいえ| K[期限切れ時にメール通知を受信]
    K --> G
    J --> L[必要に応じて CSV エクスポート]
```

## 変更点サマリー

| 項目 | As-Is | To-Be |
|-----|-------|-------|
| タスク管理方法 | スプレッドシート・付箋・メモ（個人ごとに異なる） | Greenello ボードに一元管理 |
| タスク管理場所 | 個人ごとにバラバラ | Greenello ボードに一元化 |
| 進捗確認 | 口頭・メールで都度確認 | カンバン / リスト / カレンダーで即時確認 |
| ステータス更新 | 手動で書き換え | Drag & Drop で直感的に更新 |
| 期限管理 | 見落としが発生しやすい | 期限切れタスクをメールで自動通知 |
| データ出力 | 手作業でフォーマット整形 | 任意条件で CSV エクスポート |
