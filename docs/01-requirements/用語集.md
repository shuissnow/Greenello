# 用語集

| 用語 | 定義 | 補足 |
|-----|------|------|
| タスク | ユーザーが管理する作業単位 | タスクID・タイトル・ステータス・優先度・期限・カテゴリ・担当者・作成日時・更新日時を持つ |
| サブタスク | 親タスクに紐付く子タスク | 親子関係を持つタスクの子側を指す |
| ステータス | タスクの進捗状態 | 初期リリース時点では **Inbox・進行中・完了** の3種類固定。カスタムステータスは対象外 |
| Inbox | 新規作成直後のタスクに割り当てられる初期ステータス | 未着手・未分類のタスクを一時的に受け取る場所 |
| カンバン | タスクをステータス別のカラムに分けて表示するボード形式 | Trello に代表される視覚的なタスク管理手法 |
| カラム | カンバン上のステータス区分を表す縦列 | 例：Inbox・進行中・完了 |
| Drag & Drop | タスクカードをマウスでドラッグして別のカラムにドロップする操作 | ステータス変更や完了操作に使用する |
| 優先度 | タスクの重要度・緊急度のレベル | **高・中・低** の3段階固定 |
| カテゴリ | タスクを分類するためのラベル | ユーザーが自由入力で設定する。プリセット選択肢なし。最大20文字（機能要件定義書 UC-001 参照） |
| 担当者 | タスクに割り当てられたボードメンバー | ボードメンバーから選択して設定。未設定も可。担当者でのフィルタリングが可能 |
| フィルター | 条件を指定してタスク一覧を絞り込む機能 | キーワード・期限・優先度・カテゴリで絞り込み可能 |
| CSV エクスポート | タスクデータを CSV ファイルとして出力する機能 | 出力前にフィルター条件を指定可能 |
| セッション管理 | サーバーサイドでログイン状態を管理する認証方式 | Cookie とサーバーセッションを用いる |
| 進捗サマリー | タスクの進捗状況をグラフ等で可視化する画面 | 表示形式の一つ |
| ホーム画面 | ログイン後のランディングページ。参加中のボード一覧を表示する | グローバルナビゲーションからアカウント設定画面へ遷移可能 |
| ボード | 複数ユーザーが共有するタスク管理の単位 | 1ボードに複数のカラム（ステータス）とタスクを持つ |
| ボードオーナー | ボードを作成したユーザー | ボードの管理権限（削除）を持つ |
| タスク詳細モーダル | タスクの作成・編集・削除・サブタスク管理・担当者アサインを行うポップアップ画面 | 各表示モード（カンバン・リスト・カレンダー）からタスクカードをクリックして開く |
| アカウント設定画面 | パスワード変更を行う画面 | グローバルナビゲーションからどの画面でも遷移可能 |
| ボード設定画面 | ボード削除を行う管理画面 | ボードオーナーのみアクセス可能 |
| RESTful API | HTTP メソッドとリソース URL に基づく Web API 設計スタイル | バックエンドの API 設計に採用 |
| 期限切れ通知 | 期限日を過ぎても未完了のタスクをメールで知らせる機能 | バックグラウンドジョブが毎日定刻に検出・送信する |
| バックグラウンドジョブ | ユーザー操作とは非同期に定期実行されるサーバー処理 | 期限切れタスクの検出・メール送信に使用 |
