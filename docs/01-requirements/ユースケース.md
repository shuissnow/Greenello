# ユースケース

## 機能一覧

| 機能ID | 機能名 | 概要 | 優先度 | 対応UC |
|-------|-------|------|-------|-------|
| F-001 | ユーザー認証 | 新規登録・ログイン・ログアウト・パスワード変更を含むセッション管理 | 高 | UC-006, UC-009, UC-010, UC-011 |
| F-002 | タスク作成 | タイトル・カテゴリ・期限・優先度・担当者を入力してタスクを追加 | 高 | UC-001 |
| F-003 | タスク一覧表示 | カンバン・リスト・カレンダー・進捗サマリーの4形式で表示 | 高 | 個別UCなし（表示切替はボード画面内で実現） |
| F-004 | タスク編集 | タイトル・優先度・カテゴリ・期限・担当者などを個別に更新 | 高 | UC-012 |
| F-005 | タスク完了 | チェックボックスまたは Drag & Drop で完了ステータスへ変更 | 高 | UC-003 |
| F-006 | タスク削除 | タスクを永続的に削除する | 高 | UC-013 |
| F-007 | ステータス管理 | Drag & Drop でカンバンのカラム間を移動してステータス更新 | 高 | UC-002 |
| F-008 | タスク検索・フィルター | キーワード・期限・優先度・カテゴリ・担当者で絞り込み | 中 | UC-004 |
| F-009 | 親子タスク管理 | タスクにサブタスクを紐付け、階層関係を可視化。**2階層のみ**（親→サブタスク）。サブタスクへのサブタスク追加は UI レベルで非表示にして不可とする | 中 | 個別UCなし（タスク作成・編集UC内で実現） |
| F-010 | CSV エクスポート | 表示中のタスクをフィルター指定して CSV 出力 | 中 | UC-005 |
| F-012 | 期限切れメール通知 | 期限を過ぎても未完了のタスクをメールでユーザーに通知する | 中 | UC-008 |
| F-013 | 担当者アサイン | タスクにボードメンバーを担当者として設定する | 中 | UC-014 |
| F-014 | ボード削除 | ボードオーナーがボードとその全タスクを削除する | 中 | UC-015 |

## ユースケース詳細

### UC-001: タスクを作成する

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み
- **主フロー**:
  1. ユーザーが「タスク追加」ボタンを押す
  2. タイトルを入力する（必須）
  3. 任意でカテゴリ・期限・優先度・担当者を入力する
  4. 「保存」を押すと Inbox ステータスでタスクが作成される
- **代替フロー**: タイトル未入力の場合はエラーメッセージを表示し保存しない
- **事後条件**: タスクが Inbox カラムに表示される
- **入力バリデーション**:
  - タイトル: 必須・最大100文字
  - カテゴリ: 任意・最大20文字・自由入力
  - 期限: 任意・YYYY-MM-DD 形式（JST 基準）・過去日の設定も可能
  - 優先度: 任意・高・中・低のいずれか（デフォルト: 未設定）
  - 担当者: 任意・ボードメンバーから選択
- **表示モード別の初期値**:
  - カンバン・リスト表示から作成: ステータス=Inbox、期限=未設定
  - カレンダー表示から作成: ステータス=Inbox、期限=クリックした日付を自動入力

### UC-002: タスクのステータスを変更する

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み・1件以上のタスクが存在する
- **主フロー**:
  1. ユーザーがカンバン上のタスクカードをドラッグする
  2. 目的のカラムにドロップする
  3. タスクのステータスが更新される
- **代替フロー**: カラム外へのドロップ、または Inbox・進行中・完了 以外のカラムへのドロップの場合は元の位置に戻す
- **事後条件**: タスクが新しいステータスで保存・表示される

### UC-003: タスクを完了にする

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み・タスクが存在する
- **主フロー**:
  1. タスクのチェックボックスを押す、または「完了」カラムにドロップする
  2. タスクタイトルに打ち消し線が付く
  3. タスクが完了状態として保存される
- **代替フロー**: なし
- **事後条件**: タスクが完了ステータスになり、打ち消し線で表示される

### UC-004: タスクを検索・フィルターする

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み
- **主フロー**:
  1. 検索バーにキーワードを入力、またはフィルター条件（期限・優先度・カテゴリ・担当者）を選択する
  2. 条件に一致するタスクのみ表示される
- **代替フロー**: 条件に一致するタスクがない場合は「該当なし」を表示
- **事後条件**: 絞り込まれたタスク一覧が表示される

### UC-005: CSV をエクスポートする

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み
- **主フロー**:
  1. エクスポート対象のフィルター条件を指定する
  2. 「CSV 出力」ボタンを押す
  3. CSV ファイルがダウンロードされる
- **代替フロー**: 対象タスクが 0 件の場合は警告を表示
- **事後条件**: タスクデータを含む CSV ファイルがダウンロードされる
- **出力仕様**: 出力フィールドはタスクID・タイトル・ステータス・優先度・カテゴリ・担当者・期限・作成日時・更新日時。文字コードは UTF-8（BOM付き）。カラムヘッダーは日本語。サブタスクは親タスクとは別行で出力し、「親タスクID」列に親のタスクIDを記載する。

### UC-006: ユーザーを登録する

- **アクター**: 未登録ユーザー
- **事前条件**: 未ログイン状態
- **主フロー**:
  1. ログイン画面の「新規登録」リンクを押す
  2. メールアドレスとパスワードを入力する
  3. 「登録」を押すと自動ログインしてホーム画面に遷移する
- **代替フロー**:
  - メールアドレスが既に登録済みの場合はエラーメッセージを表示
  - パスワードポリシー（非機能要件定義書 セキュリティ参照）を満たさない場合はエラーメッセージを表示
- **事後条件**: アカウントが作成され、ログイン済み状態でホーム画面が表示される

### UC-008: 期限切れタスクの通知メールを受け取る

- **アクター**: 登録ユーザー
- **事前条件**: 期限が設定されており、期限日を過ぎても完了ステータスでないタスクが存在する
- **主フロー**:
  1. バックグラウンドジョブが毎日定刻（08:00 JST）に期限切れタスクを検出する
  2. 対象タスクの作成者および担当者の登録メールアドレス宛に通知メールを送信する（同一ユーザーが作成者と担当者を兼ねる場合は1通のみ送信）
  3. メールにはタスクタイトル・期限日・ボード名・タスク詳細へのリンクを含める
- **代替フロー**: 対象タスクが 0 件の場合はメールを送信しない
- **事後条件**: ユーザーが期限切れタスクの存在をメールで確認できる
- **通知対象の条件**: ステータスが「完了」以外（Inbox または進行中） かつ 期限日 < 当日 かつ 期限超過から30日以内（30日以上放置されたタスクへの毎日通知を抑制し、通知の実用性を維持するため）

### UC-009: パスワードを変更する

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み
- **主フロー**:
  1. アカウント設定画面の「パスワード変更」を押す
  2. 現在のパスワードと新しいパスワード（確認入力含む）を入力する
  3. 「変更」を押すとパスワードが更新される
- **代替フロー**:
  - 現在のパスワードが一致しない場合はエラーを表示
  - 新しいパスワードがポリシー（非機能要件定義書 セキュリティ参照）を満たさない場合はエラーを表示
  - 新しいパスワードと確認入力が一致しない場合はエラーを表示
- **事後条件**: パスワードが更新され、既存セッションはすべて無効化される

### UC-010: ログインする

- **アクター**: 登録ユーザー
- **事前条件**: 未ログイン状態
- **主フロー**:
  1. ログイン画面でメールアドレスとパスワードを入力する
  2. 「ログイン」を押すとセッションが開始され、ホーム画面に遷移する
- **代替フロー**:
  - メールアドレスまたはパスワードが誤っている場合はエラーメッセージを表示（どちらが誤りかは明示しない）
  - 連続10回ログイン失敗した場合はアカウントを一時ロック（30分間）し、その旨を表示する
- **事後条件**: ログイン済み状態でホーム画面が表示される

### UC-011: ログアウトする

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み
- **主フロー**:
  1. グローバルナビゲーションの「ログアウト」を押す
  2. サーバー側のセッションが破棄される
  3. ログイン画面に遷移する
- **代替フロー**: なし
- **事後条件**: セッションが破棄され、未ログイン状態に戻る

### UC-012: タスクを編集する

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み・編集対象のタスクが存在する
- **主フロー**:
  1. タスクカードをクリックしてタスク詳細モーダルを開く
  2. タイトル・カテゴリ・期限・優先度・担当者を編集する
  3. 「保存」を押すと変更が反映される
- **代替フロー**: タイトルを空にして保存しようとした場合はエラーメッセージを表示し保存しない
- **事後条件**: 変更内容が保存され、タスク一覧に反映される
- **入力バリデーション**: UC-001 と同様

### UC-013: タスクを削除する

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み・削除対象のタスクが存在する
- **主フロー**:
  1. タスク詳細モーダルの「削除」ボタンを押す
  2. 確認ダイアログが表示される
  3. 「OK」を押すとタスクが削除される
- **代替フロー**:
  - 確認ダイアログで「キャンセル」を押した場合は削除せずモーダルに戻る
  - 削除対象がサブタスクを持つ親タスクの場合は「サブタスクも含めて削除されます」と警告表示した上で削除する（サブタスクも連鎖削除）
- **事後条件**: タスク（およびサブタスク）が完全に削除され、一覧から消える

### UC-014: タスクに担当者をアサインする

- **アクター**: 登録ユーザー
- **事前条件**: ログイン済み・対象タスクが存在する・ボードにメンバーが2名以上いる
- **主フロー**:
  1. タスク詳細モーダルの担当者フィールドを押す
  2. ボードメンバーの一覧が表示される
  3. 担当者を選択して保存する
- **代替フロー**:
  - 担当者を未設定にすることも可能（空欄で保存）
- **事後条件**: タスクに担当者が設定され、担当者はタスク一覧でフィルタリング可能になる

### UC-015: ボードを削除する

- **アクター**: ボードオーナー
- **事前条件**: ログイン済み・削除対象のボードが存在する
- **主フロー**:
  1. ボード設定画面の「ボードを削除」を押す
  2. 「ボードとすべてのタスクが削除されます。この操作は取り消せません」と確認ダイアログを表示する
  3. ボード名を入力して確認する（誤操作防止）
  4. 「削除」を押すとボードとその全タスク・サブタスクが削除される
- **代替フロー**:
  - 入力したボード名が一致しない場合はエラーを表示し削除しない
  - 確認ダイアログをキャンセルした場合は削除せず設定画面に戻る
- **事後条件**: ボードとその全リソース（タスク・サブタスク）が削除され、ホーム画面に遷移する
